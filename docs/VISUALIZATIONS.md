# ROS2 Multi-AMR Factory Navigation - Visualization Guide

This document indexes all visualization images generated by the demo suite. Each image illustrates key features of the fleet navigation system.

## Overview

The demonstration suite generates 7 high-quality PNG visualizations covering:
- Factory layout and environment
- AMR kinematics and motion
- Path planning algorithms
- Docking sequences
- Multi-robot fleet coordination
- Obstacle avoidance
- Alert system and real-time coordination

### 7. Alert System Demo - Multi-Robot Coordination (`alert_demo.png`)
**Size:** 210 KB | **Resolution:** 1500x1000 pixels

**NEW** - Complex visualization of fleet-wide alert broadcasting and dynamic rerouting.

**Fleet Configuration:**

#### Robot 1 (Blue) - Normal Operation
- **Start:** (15, 20) → **Goal:** (75, 25)
- **Status:** NAVIGATING (green)
- **Action:** Continuous forward progress
- **Progress:** 35m traveled at t=3.5s

#### Robot 2 (Red) - Alert Generator
- **Start:** (15, 40) → **Goal:** (75, 40)
- **Obstacle:** Person/equipment at (45, 40)
- **Status:** E-STOP (red)
- **Action Sequence:**
  - t=0s: Start navigation
  - t=3.0s: Detect obstacle → Emergency stop
  - t=3.1s: Broadcast alert to hub
  - t=3.2s: Alert relayed to fleet
  - t=5.0s: Obstacle clears, resume
- **Distance to Obstacle:** 38m→9m→0m (stopped)

#### Robot 3 (Green) - Alert Receiver
- **Start:** (15, 60) → **Original Goal:** (75, 55)
- **Status:** REROUTED (green)
- **Action Sequence:**
  - t=0s: Start navigation
  - t=3.5s: Receive Robot 2's alert
  - t=3.5s: Calculate alternate route
  - t=3.5s+: Navigate around exclusion zone
- **New Path:** Detour +5m north to avoid danger area

**Alert Timeline:**

| Time | Event | Robot | Message |
|------|-------|-------|---------|
| 0.0s | Start | All | Robots navigating |
| 3.0s | Detect | R2 | Obstacle detected! |
| 3.1s | Alert Send | R2 | Alert sent to hub |
| 3.2s | Broadcast | Hub | Broadcast to fleet |
| 3.5s | Alert Receive | R3 | Alert received, rerouting |
| 5.0s | Resume | R2 | Obstacle cleared, resuming |

**System Features:**
- **Centralized Coordination:** Hub-based alert management
- **No Direct Communication:** Robots don't communicate peer-to-peer
- **Dynamic Rerouting:** Immediate path adjustment upon alert
- **Exclusion Zones:** 3.0m radius safety area around obstacles
- **Collision-Free:** Zero collisions despite close proximity

**Alert Message Example:**
```
ALERT SYSTEM STATUS:
t=3.0s: Robot 2 detects obstacle
t=3.1s: Alert broadcasted to hub
t=3.2s: Alert sent to all robots
t=3.5s: Robot 3 receives & reroutes
t=5.0s: Obstacle cleared

Result: Zero collision, optimal avoidance
```

## Running the Demo Suite

### Generate All New Visualizations
```bash
python3 main.py --demo navigation
python3 main.py --demo obstacles
python3 main.py --demo alerts
```

### Generate All Visualizations
```bash
python3 main.py --demo all
```
